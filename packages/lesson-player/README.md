# @ssi/lesson-player

Drop-in marketing-demo custom element for the SaySomethingin landing page. Renders a ~3-minute "first full seed" lesson from any of 11 paid courses (Big 10 minus English, plus Welsh N/S) pulled from pre-built demo manifests.

**Zero framework assumptions.** Shadow DOM. Self-contained JS bundle. Works in React, Next, Vue, plain HTML — anywhere a custom element works.

## Quick start

```html
<script type="module" src="/path/to/ssi-lesson-player.js"></script>

<ssi-lesson-player
  course="spa_for_eng"
  autoplay="false"
  manifest-src="/path/to/manifests/"
  audio-src="https://saysomethingin.app/api/audio/">
</ssi-lesson-player>
```

## Attributes

| Attribute     | Type      | Default                                       | Purpose |
|---------------|-----------|-----------------------------------------------|---------|
| `course`      | string    | _required_                                    | Course code. Must have a matching manifest at `<manifest-src>/<course>.json`. |
| `autoplay`    | `"true"` \| `"false"` | `"false"`                       | Start playing on load. Off by default per best practice; explicit user click starts audio. |
| `manifest-src`| URL       | `./manifests/` (relative to the script URL)   | Base URL where `<course>.json` and `index.json` live. |
| `audio-src`   | URL       | `https://saysomethingin.app/api/audio/`       | Base URL for the audio proxy. Audio IDs are appended directly. |

## Sizing

- **Fluid width**, fixed aspect (440 × 900 roughly).
- `max-width: 440px`, `aspect-ratio: 440 / 900`.
- Shrink the container width and the whole player scales proportionally.
- To center in a section: wrap in a flex container, done.

## Demo mode (what the element does and doesn't do)

- **Does:** plays a 4-phase cycle (prompt → pause → voice 1 → voice 2) for each item, shows timer, shows belt progress, shows the end-of-demo CTA.
- **Does not:** request microphone, require login, touch production Supabase, send telemetry, require Vercel-hosted audio (plain S3 signed URLs via the proxy work fine).

## Regenerating manifests

When course content changes in the production DB:

```bash
# From the repo root
cd packages/lesson-player
node scripts/build-manifests.cjs
# Manifests written to public/manifests/*.json
```

The script reads from the dashboard `.env` (service key) and writes 11 JSON files. Re-run this after any seed 1 edit or audio regeneration.

## Local preview

```bash
cd packages/lesson-player
npm run serve
# Open http://localhost:8765/demo.html
```
